## 1. 岗位职责概述

算法工程师是动车组检修数字孪生仿真系统（EMU-DT-SIM）的核心技术岗位，负责系统中所有算法模型的设计、开发、训练、部署与持续优化。本岗位涵盖调度优化、路径规划、PHM预测性维护、AI视觉检测、仿真引擎集成等多个技术方向。

### 1.1 核心职责

| 职责领域 | 职责描述 | 对应需求编号 |
|---------|---------|------------|
| **调度优化算法** | 负责段厂调车调度优化、检修台位分配优化、资源调度优化及动态事件自适应调度算法的设计、实现与调优 | AR-001~AR-004 |
| **路径规划算法** | 负责段厂内调车路径规划、动态避障路径规划及多车协同路径规划算法的开发 | AR-005~AR-007 |
| **PHM故障预测** | 负责设备RUL预测模型、故障诊断模型、异常检测模型及退化趋势预测模型的训练与部署 | AR-011~AR-014 |
| **AI视觉检测** | 负责部件缺陷检测模型、表面缺陷分割模型、三维点云处理算法及磨耗测量算法的训练与优化 | AR-015~AR-018 |
| **仿真引擎集成** | 负责离散事件仿真引擎、物理仿真引擎及工艺流程仿真引擎的集成与调优 | AR-008~AR-010 |
| **数字孪生同步** | 负责状态估计算法和多源数据融合算法的开发，实现数字孪生模型与物理实体的精确同步 | AR-019~AR-020 |
| **生产调度排程** | 负责约束规划求解器、混合优化算法及滚动窗口动态调度算法的开发 | AR-021~AR-023 |

### 1.2 协作关系

| 协作对象 | 协作内容 |
|---------|---------|

### 1.3 技能要求

- **数学基础**：运筹学、概率统计、最优化理论、图论
- **机器学习**：深度学习（CNN/RNN/Transformer）、强化学习、无监督学习
- **编程语言**：Python（主要）、C#（Unity集成）、C++（性能关键模块）

#### AR-001 段厂调车调度优化算法

| 属性 | 内容 |
|------|------|
| **算法名称** | 段厂调车调度优化算法 |
| **算法类型** | 混合元启发式算法（GA+SA+PSO） |
| **适用场景** | 段厂内动车组入库、转线、出库等调车作业的全局调度优化 |
| **输入** | 调车任务列表、股道拓扑、动车组位置、台位占用状态、时间约束 |
| **输出** | 最优调车序列、各车调车路径、预计完成时间、资源冲突检测结果 |
| **性能指标** | 求解时间≤30秒（100个调车任务规模），方案质量偏差≤5% |
| **优先级** | P0 |

**算法设计要点：**

1. **遗传算法（GA）模块**：负责全局搜索，编码方式采用整数排列编码，染色体表示调车任务执行序列
2. **模拟退火（SA）模块**：负责局部搜索优化，在GA产生的新解基础上进行精细调整
3. **粒子群优化（PSO）模块**：负责多目标优化，平衡调车时间、资源利用率和能耗
4. **约束处理**：股道容量约束、道岔互斥约束、安全间隔约束、时间窗口约束
5. **动态调整**：支持突发任务插入时的快速重调度

        track_topology: 股道拓扑结构
        emu_positions: 动车组当前位置
        station_status: 台位占用状态
        time_constraints: 时间约束

        DispatchSolution: 包含最优调车序列、路径、预计时间

#### AR-002 检修台位分配优化算法

| 属性 | 内容 |
|------|------|
| **算法名称** | 检修台位分配优化算法 |
| **算法类型** | 整数规划/约束规划 |
| **适用场景** | 检修台位与动车组的匹配分配优化 |
| **输入** | 检修工单列表、台位信息、台位能力、时间窗口、约束条件 |
| **输出** | 最优台位分配方案、各台位作业排程、利用率预估 |
| **性能指标** | 求解时间≤10秒（50个工单规模），台位利用率提升≥10% |
| **优先级** | P0 |

**算法设计要点：**

1. **决策变量**：二元决策变量 x[i][j] 表示工单i是否分配到台位j
2. **目标函数**：最小化总检修时间 + 最大化台位利用率
3. **约束条件**：
   - 每个工单恰好分配到一个台位
   - 同一台位同一时间仅处理一个工单
   - 台位能力匹配（台位类型与修程类型匹配）
   - 时间窗口约束（工单必须在规定时间窗口内完成）
4. **求解策略**：优先使用精确求解（OR-Tools CP-SAT），大规模问题切换到启发式求解

### 2.2 路径规划算法需求

#### AR-005 段厂内调车路径规划

| 属性 | 内容 |
|------|------|
| **算法名称** | 段厂内调车路径规划算法 |
| **算法类型** | A*算法（基于轨道拓扑图） |
| **适用场景** | 动车组在段厂股道间的调车路径规划 |
| **输入** | 起点位置、终点位置、股道拓扑、道岔状态、占用信息 |
| **输出** | 最优调车路径、路径长度、预计时间、道岔操作序列 |
| **性能指标** | 求解时间≤1秒，路径最优性偏差≤10% |
| **优先级** | P0 |

**算法设计要点：**

1. **图模型构建**：将段厂股道拓扑建模为有向加权图，节点为道岔/信号机位置，边为股道段
2. **A*搜索**：采用A*算法在拓扑图上搜索最短路径，启发函数使用欧几里得距离
3. **动态权重**：考虑道岔切换代价、股道占用状态、限速区段
4. **路径平滑**：对搜索结果进行路径平滑处理，确保路径物理可行性
5. **冲突检测**：实时检测路径上的资源冲突，支持重规划

#### AR-007 多车协同路径规划

| 属性 | 内容 |
|------|------|
| **算法名称** | 多车协同路径规划算法 |
| **算法类型** | 多智能体路径规划（CBS/MAPF） |
| **适用场景** | 多辆动车组同时调车时的无冲突路径规划 |
| **输入** | 多车起点终点、股道拓扑、时间约束 |
| **输出** | 多车无冲突路径集、时间调度方案 |
| **性能指标** | 求解时间≤5秒（5车规模），无冲突保证 |
| **优先级** | P1 |

**算法设计要点：**

1. **CBS（Conflict-Based Search）**：基于冲突的搜索算法，在高层搜索冲突解决策略
2. **优先级规划**：为每辆车分配优先级，按优先级依次规划路径
3. **时空A***：在时空状态空间中搜索，避免时间维度的冲突
4. **死锁检测**：实时检测多车调车中的死锁状态并提供解锁策略
5. **规模扩展**：大规模场景下采用分层规划策略

### 2.3 仿真引擎需求

#### AR-008 离散事件仿真引擎

| 属性 | 内容 |
|------|------|
| **算法名称** | 离散事件仿真引擎（DES） |
| **算法类型** | 事件驱动仿真 |
| **适用场景** | 段厂检修作业流程仿真、调车作业仿真、资源调度仿真 |
| **输入** | 仿真模型定义、事件列表、资源模型、初始状态 |
| **输出** | 仿真过程数据、关键指标统计、瓶颈分析、资源利用率 |
| **性能指标** | 仿真速度≥100倍实时速度，支持≥1000个并发事件 |
| **优先级** | P0 |

**算法设计要点：**

1. **事件队列**：采用优先队列（最小堆）管理事件，按时间戳排序
2. **事件类型**：任务开始、任务完成、资源占用、资源释放、设备故障等
3. **时间推进**：采用下一事件推进策略，跳过无事件时间段
4. **随机数**：支持多种概率分布（均匀、正态、指数、三角分布）
5. **统计收集**：自动收集资源利用率、等待时间、吞吐量等关键指标
6. **技术选型**：SimPy（Python）用于算法开发与验证，Unity C#侧实现高性能版本

1. **出入库路径优化**：基于MIP模型优化堆垛机行走路径，最小化总运行时间
2. **货位分配优化**：基于货物出入库频率和关联性进行货位分配优化（高频靠近出入口、关联货物集中存放）
3. **堆垛机调度优化**：多台堆垛机的任务分配和执行顺序优化，避免巷道冲突
4. **动态调度**：支持紧急出入库任务的动态插入和调度调整
5. **与检修联动**：考虑检修作业物料需求时间窗的联动优化

### 2.12 仿真数据生成算法需求

#### AR-028 仿真数据生成算法

| 属性 | 内容 |
|------|------|
| **算法名称** | 仿真数据生成算法 |
| **算法类型** | 蒙特卡洛模拟 + 时序数据生成模型 |
| **适用场景** | 设备运行数据和车辆状态数据的仿真生成，用于系统测试和培训 |
| **输入** | 数据生成配置（分布类型、参数范围、时间范围、故障模式等） |
| **输出** | 仿真时序数据、故障事件序列、状态变迁记录 |

### 3.3 各环节技术方案详细说明

#### AI-001 车顶设备检查

- **检测目标**：受电弓滑板磨损、绝缘子裂纹/污损、天线变形
- **模型方案**：YOLOv8目标检测定位设备位置，ResNet分类判断状态
- **输入规格**：1920x1080可见光图像，支持红外图像
- **输出规格**：设备位置边界框、状态分类（正常/异常）、置信度
- **准确率要求**：检测mAP@0.5≥90%，分类准确率≥95%

#### AI-002 车轮踏面检查

- **检测目标**：踏面擦伤、剥离、裂纹、磨耗超限
- **模型方案**：U-Net语义分割精确定位缺陷区域，AutoEncoder检测异常
- **输入规格**：踏面高分辨率图像（≥4000x3000），激光轮廓数据
- **输出规格**：缺陷分割掩码、缺陷类型、磨耗测量值
- **准确率要求**：缺陷检测召回率≥95%，磨耗测量精度≤0.1mm

#### AI-003 制动盘检查

- **检测目标**：制动盘裂纹、偏磨、热斑
- **模型方案**：U-Net语义分割检测裂纹和偏磨区域，Isolation Forest检测热斑异常
- **输入规格**：制动盘表面图像，红外热成像数据
- **输出规格**：裂纹位置和长度、偏磨量、热斑区域
- **准确率要求**：裂纹检测IoU≥85%，偏磨测量精度≤0.2mm

#### AI-004 转向架检查

- **检测目标**：构架裂纹、螺栓松动、减振器漏油
- **模型方案**：YOLOv8目标检测定位关键部件，AutoEncoder检测异常状态
- **输入规格**：转向架多角度图像
- **输出规格**：部件状态（正常/异常）、异常位置、异常类型
- **准确率要求**：检测准确率≥90%，异常检测召回率≥95%

#### AI-005 车底设备检查

- **检测目标**：牵引电机状态、齿轮箱状态、制动夹钳状态
- **模型方案**：YOLOv8目标检测定位设备，ResNet分类判断状态
- **输入规格**：车底巡检机器人采集图像
- **输出规格**：设备状态分类、异常标记
- **准确率要求**：状态分类准确率≥92%

#### AI-006 车内设施检查

- **检测目标**：座椅损坏、门窗异常、卫生间设施状态
- **模型方案**：YOLOv8目标检测 + MobileNet轻量分类（移动端部署）
- **输入规格**：车内巡检图像
- **输出规格**：设施状态分类、损坏位置标记
- **准确率要求**：分类准确率≥90%

#### AI-007 油液状态检查

- **检测目标**：齿轮箱油位、油色异常、油质劣化
- **模型方案**：传统图像处理（颜色直方图+HSV分析）+ 轻量分类模型
- **输入规格**：油液观察窗图像
- **输出规格**：油位正常/异常、油色等级、油质评估
- **准确率要求**：油色识别准确率≥95%

#### AI-008 接地碳刷检查

- **检测目标**：碳刷长度、磨损状态、弹簧状态
- **模型方案**：传统CV测量算法（轮廓提取+尺寸计算）+ 异常检测
- **输入规格**：碳刷特写图像
- **输出规格**：碳刷剩余长度、磨损等级、弹簧状态
- **准确率要求**：长度测量精度≤0.5mm

## 4. 设备故障预警算法需求

### 4.1 功能描述

设备故障预警功能基于设备运行数据和AI算法，对设备潜在故障进行预测和预警，实现从事后维修向预测性维护的转变。

### 4.2 阈值预警（FR-203-01）

| 属性 | 内容 |
|------|------|
| **需求编号** | FR-203-01 |
| **优先级** | P0 |
| **算法职责** | 定义阈值预警规则引擎，支持多维度参数阈值配置 |

**算法实现要点：**

1. **固定阈值预警**：基于设备出厂参数和历史运行数据设定固定阈值
2. **动态阈值**：根据设备运行工况（负载率、环境温度等）动态调整阈值
3. **多参数联合预警**：支持多参数组合条件触发预警
4. **迟滞处理**：避免阈值附近的频繁报警，设置迟滞区间
5. **预警分级**：根据超限程度自动分级（提示/警告/严重/紧急）

2. **变化率检测**：计算参数变化速率，检测加速退化
3. **滑动窗口统计**：基于滑动窗口计算均值、标准差、变化率
4. **预测外推**：基于趋势拟合结果外推预测参数到达阈值的时间

**评估流程：**

1. **时序划分**：按时间顺序划分训练/验证/测试集（避免数据泄露）

3. **生存分析**：使用C-index评估RUL预测的判别能力
4. **在线评估**：部署后持续监控模型预测效果
